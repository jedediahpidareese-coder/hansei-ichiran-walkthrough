22  Interactive map (prototype)

Each polygon is a daimyō (大名) domain (藩) from 1870. You can toggle what the colours mean using the layer control in the top-right of the map.

TipHow to use this map

Five layers to toggle between (radio buttons, top-right corner):

  1. 1884 kazoku (華族) Rank (default) — colour = peerage rank received by the family that ruled each domain in 1870
  2. kokudaka (石高) — colour = composite revenue assessment on a log scale (the paper’s main predictor)
  3. Boshin Alignment — colour = which side the domain’s leadership took in the 1868–69 civil war
  4. Tokugawa-era Category — colour = shinpan (親藩) / fudai (譜代) / tozama (外様) / sankyō (御三卿) / tenryō (天領)
  5. Genmai-Rule Deviations — only the 19 paper-critical cases are coloured; everything else fades to grey

Other controls:

  • Jump-to-domain search (top-left corner): type a domain name — romanized or kanji — and the map flies straight to it
  • EN / 日本語 toggle (bottom-right corner): switches all popup contents + legend text between English and Japanese
  • Hover any polygon for a quick tooltip
  • Click any polygon for a full data card showing: the family that ruled it, the 1884 rank recipient (with citation), all the kokudaka/population/military/Boshin values, sources for each, and — for the four 薩長土肥 focal domains — links out to the scholarly literature
  • Pan + zoom with mouse or trackpad
ImportantThe family-vs-domain framing applies throughout

When you click a polygon and see “this family received marquess in 1884,” the literal claim is: the individual head of the family that ruled this domain in 1870 was awarded marquess rank from the Imperial Court in 1884. The domain itself had been abolished in 1871. See kazoku Rank for the full temporal sequence.

NoteThe blank white areas

Large stretches of the map — most visibly through central Honshū around Kōfu — are blank white with no polygon. That is land which was not a daimyō domain: shogunal direct land (tenryō 天領), bannermen (hatamoto 旗本) holdings, and temple / shrine / court land, which together were roughly a quarter of Edo-period Japan. None of it had a daimyō, so none of it is in a 286-domain dataset. See the territorial-changes chapter for the full explanation.

NoteWhat you’re seeing

Each colored polygon is one of the 286 daimyō domains the project tracks. The polygon shapes come from the Harvard CHGIS 1664 daimyō shapefile — they are an approximation of 1870 boundaries (which were generally stable but had some changes). Polygons that didn’t match a project domain are shown in grey.

The data behind each popup is the same data as the static maps — the column from analytic_wide.csv. The popup card shows:

  • Revenue — composite kokudaka + genmai (現米)
  • Demographics — population, samurai (侍) class breakdown (Meiji Shiyō)
  • Military — total troops, warship count (Hansei Ichiran 1870 vol. 2 where available)
  • Classification — Tokugawa-era category, Boshin alignment
  • 1884 kazoku rank — actual + expected (kokudaka-rule + genmai-rule) + deviation
  • Sources — every citation for the row’s data

22.1 What the map can do

The map is a single self-contained HTML file — it works offline, needs no server, and embeds in this walkthrough as readily as it opens on its own. It offers:

  • Five colour layers — 1884 kazoku rank, kokudaka, Boshin alignment, Tokugawa-era category, and the 19 genmai-rule deviations — switched from the layer control in the top-right corner.
  • A jump-to-domain search box (top-left) — type a domain in romanized or kanji form to fly straight to it.
  • An EN / 日本語 toggle — switches every popup and the legend between English and Japanese.
  • Click-through data cards — every one of the 286 domains opens a full card: the ruling family, the 1884 rank recipient, revenue, demographics, military, Boshin alignment, the rank-deviation diagnostic, and a per-row source list. The four 薩長土肥 focal domains also link out to their walkthrough chapters and key scholarly works.

Every value in a popup is the same data as the static choropleth maps elsewhere in the walkthrough — the interactive map simply lets you query it domain by domain.

Implementation: Folium 0.20 (a Python wrapper for Leaflet.js). Polygon source: Harvard CHGIS 1664 daimyō shapefile. Data: analytic_wide.csv (286 × 107). Output: a self-contained HTML map (interactive_map.html).
